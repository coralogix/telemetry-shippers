# otel-linux-standalone-govcloud

Terraform + Helm workflow to deploy a Coralogix OpenTelemetry collector on an EC2 instance in **AWS GovCloud (US)**. Based on `otel-linux-standalone`, adapted for FedRAMP/GovCloud compliance requirements.

No EKS, ECS, CloudWatch, or S3 dependencies. Pure EC2 → OTEL → Coralogix.

## Architecture

```
┌─────────────────────────────────────────────┐
│  Private Subnet (us-gov-east-1 / west-1)    │
│                                             │
│  EC2 (Ubuntu 22.04, t3.small)               │
│  └── otelcol-contrib (systemd service)      │
│        ├── journald receiver                │
│        │     └── all host systemd logs      │
│        ├── hostmetrics receiver             │
│        │     └── CPU, mem, disk, net        │
│        ├── prometheus receiver              │
│        │     └── collector self-metrics     │
│        ├── OTLP receiver (4317/4318)        │
│        │     └── app instrumentation        │
│        └── Coralogix exporter               │
│              └── ingress.coralogix.com      │  ← coralogixgov.com when live
└─────────────────────────────────────────────┘
         │ outbound via NAT gateway
```

## Prerequisites

- `helm` v3 and `yq`
- `terraform` >= 1.5.7
- AWS CLI configured with a GovCloud profile (e.g. `gov`)
- An existing VPC and subnet — GovCloud accounts typically have no default VPC (see [Optional VPC Module](#optional-vpc-module) if you need one)
- A Coralogix [Send-Your-Data API key](https://coralogix.com/docs/send-your-data-api-key/)

## Deployment

### Step 1 — Create a tfvars file

Copy the example below and fill in your values. **Do not commit this file** — it is git-ignored by `*.tfvars`.

```hcl
# terraform/terraform.tfvars

aws_region  = "us-gov-east-1"   # or us-gov-west-1
aws_profile = "gov"              # AWS CLI profile for GovCloud

vpc_id    = "vpc-xxxxxxxxxxxxxxxxx"
subnet_id = "subnet-xxxxxxxxxxxxxxxxx"  # private subnet recommended

otlp_ingress_cidrs = ["10.0.0.0/16"]   # restrict to your VPC CIDR

otel_config_path = "../build/otel-config.yaml"
instance_type    = "t3.small"

associate_public_ip_address = false

coralogix_api_key = "your-send-your-data-api-key"  # sensitive — do not commit

tags = {
  Environment = "production"
}
```

### Step 2 — Deploy

```bash
cd otel-linux-standalone-govcloud

# Renders the collector config from values.yaml, then runs terraform apply
make deploy
```

To use a custom tfvars file:

```bash
make deploy VAR_FILE=terraform/terraform.tfvars
```

### Step 3 — Verify

```bash
# Show instance ID, private IP, SSM command
make terraform-output

# Open a shell on the instance (no SSH key needed)
aws ssm start-session --target <instance-id> --profile gov --region us-gov-east-1

# Inside the session — check service status
sudo systemctl status otelcol-contrib.service --no-pager -l

# Tail live logs
sudo journalctl -u otelcol-contrib.service -f
```

### Tear down

```bash
make destroy
```

## Optional VPC Module

GovCloud accounts typically do not have a default VPC. If you need to provision one (e.g. for testing), use the bundled VPC module independently before deploying the collector.

### Deploy the VPC

```bash
cd terraform/modules/vpc

terraform init
terraform apply -var-file=dev.tfvars
```

The `dev.tfvars` is pre-configured for `us-gov-east-1`. To customise:

```hcl
# terraform/modules/vpc/dev.tfvars
aws_region  = "us-gov-east-1"
aws_profile = "gov"

availability_zones   = ["us-gov-east-1a"]
name_prefix          = "otel-govcloud-dev"
vpc_cidr             = "10.0.0.0/16"
private_subnet_cidrs = ["10.0.1.0/24"]
public_subnet_cidrs  = ["10.0.101.0/24"]
enable_nat_gateway   = true
enable_flow_logs     = true   # recommended — covers FedRAMP AU-2/AU-12
```

### Feed VPC outputs into the collector deployment

```bash
VPC_ID=$(terraform output -raw vpc_id)
SUBNET_ID=$(terraform output -json private_subnet_ids | jq -r '.[0]')

cd ../../   # back to terraform/
terraform apply \
  -var-file=dev.tfvars \
  -var vpc_id=$VPC_ID \
  -var subnet_id=$SUBNET_ID \
  -var otel_config_path=$(pwd)/../build/otel-config.yaml
```

### What the VPC module provisions

| Resource | Details |
|---|---|
| `aws_vpc` | DNS support + hostnames enabled |
| `aws_subnet` (private) | One per AZ — OTEL instance lives here |
| `aws_subnet` (public) | One per AZ — required for NAT gateway |
| `aws_internet_gateway` | Enables NAT gateway outbound path |
| `aws_nat_gateway` + EIP | Outbound internet from private subnet to CX ingestion endpoint |
| Route tables | Public → IGW, private → NAT |
| VPC Flow Logs | CloudWatch log group, 90-day retention by default |

## Key Configuration Points

### Coralogix Domain

`values.yaml` controls the target Coralogix endpoint via `global.domain`:

```yaml
global:
  domain: "coralogix.com"       # current — standard commercial endpoint for testing
  # domain: "coralogixgov.com"  # use this when the GovCloud domain is live
```

Switch back to `coralogixgov.com` once that domain is provisioned.

### Journald Log Directory

Ubuntu 22.04 writes journals to persistent storage at `/var/log/journal/` rather than the volatile `/run/log/journal/`. The `values.yaml` explicitly sets the correct path:

```yaml
config:
  receivers:
    journald:
      directory: /var/log/journal
```

Without this, the journald receiver starts cleanly but collects nothing.

### Disabled Presets (Standard otelcol-contrib Incompatibilities)

The following presets are disabled because they depend on components not included in the standard `otelcol-contrib` binary:

| Preset | Reason disabled |
|---|---|
| `systemdReceiver` | Uses a custom `systemd` receiver only available in the Coralogix collector distribution |
| `spanMetrics` | Generates `spanmetrics` connector config with `add_resource_attributes` key not valid in otelcol-contrib v0.137.0 |
| `transactions` | Same `add_resource_attributes` incompatibility |

### Network Ingress

All ingress rules are **closed by default**. Open only what's needed:

```hcl
# OTLP from application subnet
otlp_ingress_cidrs = ["10.0.1.0/24"]

# SSH from bastion — optional, prefer SSM Session Manager
ssh_ingress_cidrs = ["10.0.0.10/32"]
```

### Instance Access — SSM Session Manager (Recommended)

The IAM role attached to the instance includes `AmazonSSMManagedInstanceCore`. No SSH key needed:

```bash
aws ssm start-session --target <instance-id> --profile gov --region us-gov-east-1
```

> **Note:** Always use `--profile` explicitly rather than relying on exported environment variables (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_SESSION_TOKEN`). In GovCloud, SSM Session Manager is known to fail authentication silently when credentials are sourced from environment variables, even when other AWS CLI commands work. Using `--profile` directly resolves this.

The `terraform output ssm_session_command` will emit the correct command with `--profile` included.

To use SSH instead, provide key variables in your tfvars:

```hcl
ssh_key_name         = "my-key"
ssh_public_key_path  = "~/.ssh/id_rsa.pub"
ssh_private_key_path = "~/.ssh/id_rsa"
```

### Air-Gapped Environments

The collector `.deb` is fetched from GitHub at bootstrap time. In air-gapped GovCloud environments, mirror it to an internal S3 bucket and override:

```hcl
otel_deb_url = "https://s3.us-gov-east-1.amazonaws.com/my-artifacts-bucket/otelcol-contrib_0.137.0_linux_amd64.deb"
```

## What Data Gets Sent

The following telemetry is collected and exported automatically with no application changes:

| Signal | Source | Content |
|---|---|---|
| Logs | `journald` | All systemd service logs from the host |
| Metrics | `hostmetrics` | CPU, memory, disk, filesystem, network (10s interval) |
| Metrics | `prometheus` | Collector's own internal metrics (scrape interval 30s) |
| Entity events | `hostmetrics` → `logs/resource_catalog` | Host identity metadata (IP, OS, CPU, instance ID) |

Application traces, logs, and metrics can be sent to the collector via OTLP on `127.0.0.1:4317` (gRPC) or `127.0.0.1:4318` (HTTP).

## GovCloud vs Commercial Differences

| Item | Commercial (`otel-linux-standalone`) | GovCloud (`otel-linux-standalone-govcloud`) |
|---|---|---|
| Ubuntu AMI owner ID | `099720109477` | `513442679011` |
| Default region | `eu-west-1` | `us-gov-west-1` |
| Coralogix domain | `coralogix.com` | `coralogixgov.com` (pending) |
| Default VPC | Looked up automatically | Must be supplied via `vpc_id` |
| Subnet | Looked up from default VPC | Must be supplied via `subnet_id` |
| Public IP default | `true` | `false` |
| IAM policy ARN prefix | `arn:aws:iam` | `arn:aws-us-gov:iam` |
| IMDSv2 | Optional | Enforced (`http_tokens = "required"`) |
| Journald directory | Not set (uses volatile default) | Explicitly set to `/var/log/journal` |
| SSH key pair | Required | Optional — SSM Session Manager preferred |
| OTLP ingress | Open (`0.0.0.0/0`) | Closed by default, variable-driven |
